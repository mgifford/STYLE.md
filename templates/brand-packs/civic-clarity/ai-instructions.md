# AI enforcement rules for civic clarity

- Use token names from `tokens.css`; do not introduce parallel token names.
- Reuse documented components from `components.md` before creating new patterns.
- Keep sentence-case headings and plain-language copy.
- Preserve accessibility guardrails from `ACCESSIBILITY.md` and sustainability constraints from `SUSTAINABILITY.md`.
- If a new reusable pattern is required, document it in `components.md` in the same change.
