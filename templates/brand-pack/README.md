# Brand pack template

Use this folder as a copy-ready starter for keeping visual and writing brand consistency across projects.

## What this pack includes

- `tokens.css`: portable design token defaults for site surfaces
- `components.md`: component contracts and required usage rules
- `AGENTS-brand-snippet.md`: AI agent guardrails for brand consistency
- `STYLES-brand-snippet.md`: STYLES.md section snippet for brand portability

## How to use in a new repository

1. Copy this folder into your new repository as `brand-pack/`.
2. Merge `tokens.css` into your site stylesheet (or import it first).
3. Add the `components.md` contracts to your design docs.
4. Copy the AGENTS snippet into your `AGENTS.md`.
5. Copy the STYLES snippet into your `STYLES.md`.
6. Replace placeholder values once and keep token names stable.

## Why this works

Keeping token names, component contracts, and agent rules aligned gives humans and AI agents one consistent source of truth for brand decisions.
